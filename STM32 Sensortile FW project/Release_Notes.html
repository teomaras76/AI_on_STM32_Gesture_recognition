Release Notes for FP-AI-SENSING1 Software for STM32, Expansion for STM32Cube

Copyright 2018 STMicroelectronics

 

FP-AI-SENSING1 is an STM32 ODE function pack organized as a collection of independent threads of execution using FreeRTOS Real Time Kernel. Thanks to this package you can connect your IoT node to a smartphone using BLE, and by using a suitable AndroidT or iOST application such as the BlueMS app (BlueMS - Version 4.1.0 or higher), you can visualize real-time environmental sensor data, motion sensor data, digital microphone levels, and battery level. With this package you can record on SD card the enviromental sensor data, motion sensor data and digital microphone input. The package also allows to perform advanced functionality like real-time Activity Recognition (HAR_GMP/HAR_IGN/HAR_IGN_WSDM) or Audio Scene Classification based on neural networks. This package, together with the suggested combination of STM32 and ST devices can be used, for example, to develop wearable applications, and, more in general smart things applications. The software runs on the STM32 microcontroller and includes all the necessary drivers to recognize the devices on the STM32 Nucleo development board and expansion boards, as well as on the STEVAL-STLKT01V1 evaluation board. Power management function has been introduced to preserve the battery level for the STEVAL-STLKT01V1 evaluation board.

FP-AI-SENSING1 software features:
    • Complete middleware to build applications using:
      • inertial and environmental sensors
      • microphones signal level
      leveraging Bluetooth Low Energy for communication of sensors data
    • Includes one Middleware for Activity Recognition Algorithms based on neural networks: HAR_GMP/HAR_IGN/HAR_IGN_WSDM
    • Includes one Middleware for Audio Scene Recognition Algorithm based on neural networks
    • USB device library (for only STEVAL-STLCS01V1)
    • FatFs generic FAT file system module (for only STEVAL-STLCS01V1) provides access the storage devices such as memory card and hard disk (under MCD-ST Liberty SW License)
    • FreeRTOS Real Time Kernel/Scheduler that allows applications to be organized as a collection of independent threads of execution (under MIT open source license)
    • The FP-AI-SENSING1 Firmware could be updated Over-The-Air (FOTA) using the the Android/iOS device with the BlueMS Android/iOS application (BlueMS - Version 3.0.0 or higher)
    • Easy portability across different MCU families thanks to STM32Cube
    • Free user-friendly license terms

Important Hardware Additional Information

  • Before to connect X-NUCLEO-IKS01A2 with X-NUCLEO-CCAM02M1 expansion board through the Arduino UNO R3 extension connector, on to X-NUCLEO-IKS01A2 board remove SB25 0-ohm resistor.
  • Before to connect the board X-NUCLEO-CCA02M1 with the STM32 L4 Nucleo motherboard through the Morpho connector layout, on to X-NUCLEO-CCA02M1 board close the solder bridges SB12, SB16 and open the solder bridges SB7, SB15 and SB17.

ISSUE

  • None.

Known Limitations

  • Changing the Bluetooth exported characteristics could produce some errors for Android/iOS device. With Android remember to select the "Clear device cache" option on ST BLE Sensor Application before to make the connection to FP-AI-SENSING1
  • The Data Log may occasionally create some empty/corrupted file on SD card.
